## Responsive Considerations

### Desktop (1200px+)
- Full navigation bar with all items visible
- Multi-column layouts
- Full table views with all columns

### Tablet (768px - 1199px)
- Collapsible navigation
- Reduced table columns (priority columns only)
- Stacked card layouts

### Mobile (< 768px)
- Hamburger menu navigation
- Single column layouts
- Card-based data display instead of tables
- Simplified forms with full-width inputs

## Interaction Patterns

### Navigation
- Single-click to navigate between sections
- Breadcrumb-style back navigation on detail pages
- Active state clearly indicated in nav bar

### Tables
- Click row to view details
- Sortable columns (click header)
- Filterable via dropdown menus
- Bulk selection via checkboxes

### Forms
- Inline validation
- Required field indicators (*)
- Disabled submit until form is valid
- Clear error messaging

### Modals
- Centered overlay
- Close via X button or click outside
- Keyboard accessible (Esc to close)

---

## Summary

This wireframe document provides a comprehensive visual specification for the SLM Training Platform POC. All screens follow consistent design patterns with:

- **Table-first approach** for data-heavy views
- **Wizard-driven workflows** for complex operations
- **Clear status indicators** throughout
- **Strong cross-linking** between related entities
- **Read-only artifact views** for completed operations

The wireframes serve as a foundation for UI/UX implementation while maintaining alignment with the platform's core objectives of clarity, traceability, and operational correctness.
